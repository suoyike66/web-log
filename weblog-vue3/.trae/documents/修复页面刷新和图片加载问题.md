## 问题分析

### 1. 页面刷新问题
- **原因**：当用户未登录时，`Header.vue` 组件会调用 `getUserInfo()` 函数获取用户信息，该函数会调用 `/admin/user/info` API 接口
- **问题链**：未登录状态下，API 返回 401 错误 → 触发 `axios.js` 响应拦截器中的 401 处理逻辑 → 执行 `location.reload()` 刷新页面 → 页面重新加载后再次调用 `getUserInfo()` → 形成无限循环

### 2. 图片加载问题
- **原因**：`LazyImage.vue` 组件在图片加载失败时会设置 `error` 状态，并通过 CSS 隐藏图片
- **问题**：当图片 URL 无效或无法访问时，图片会一直显示加载动画，而不显示任何内容

## 修复方案

### 1. 修复页面刷新问题
- **方案**：修改 `axios.js` 中的响应拦截器，避免在未登录时刷新页面
- **具体修改**：
  1. 在 `axios.js` 中，当遇到 401 错误时，只执行退出登录操作，不刷新页面
  2. 这样可以避免未登录状态下的页面无限刷新问题

### 2. 修复图片加载问题
- **方案**：修改 `LazyImage.vue` 组件，改进图片加载失败的处理逻辑
- **具体修改**：
  1. 在 `LazyImage.vue` 中，添加错误占位符显示
  2. 当图片加载失败时，显示错误状态而不是完全隐藏图片
  3. 确保图片加载逻辑正确处理各种情况

## 修复步骤

1. **修改 `axios.js`**：
   - 移除 401 错误处理中的 `location.reload()` 语句
   - 保持退出登录操作，以确保用户状态正确重置

2. **修改 `LazyImage.vue`**：
   - 添加错误状态的显示逻辑
   - 确保图片加载失败时用户可以看到明确的错误提示
   - 优化图片加载逻辑，确保图片能够正确加载

## 预期效果

- **页面刷新问题**：未登录状态下页面不再无限刷新，能够正常显示首页内容
- **图片加载问题**：图片加载失败时显示错误提示，而不是一直显示加载动画
- **用户体验**：提升网站的稳定性和用户体验，避免不必要的页面刷新和图片加载问题
